# Security policy

Safe Space Cleaner performs destructive filesystem operations only after an audit plan is token-verified. Please report path-boundary bypasses, reparse-point traversal, plan-validation weaknesses, unintended deletion classes, credential exposure, or report-privacy leaks through a private GitHub security advisory rather than a public issue.

Do not attach real audit or cleanup reports to an issue. Reproduce with the isolated test harness and replace usernames, machine names, filenames, and paths with synthetic values.
